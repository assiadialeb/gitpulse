Comment ça fonctionne

1

Utilisateur se connecte

L'utilisateur se connecte via GitHub OAuth avec les scopes : user:email, repo, read:org

2

Token utilisateur stocké

Le token OAuth de l'utilisateur est stocké par django-allauth

3

Accès aux repositories

GitPulse utilise le token de l'utilisateur pour accéder aux repositories auxquels il a accès

4

Tâches en arrière-plan

Les tâches d'indexation utilisent le token de l'utilisateur propriétaire de l'application

Dépannage

Repositories inaccessibles

Si certains repositories ne sont pas accessibles :

  • Vérifiez que vous avez accès aux repositories sur GitHub
  • Pour les organisations, vérifiez que l'OAuth App est autorisée
  • Reconnectez-vous pour mettre à jour les permissions
Scopes manquants

Si vous n'avez pas tous les scopes requis :

  • Utilisez le bouton "Reconnecter" ci-dessus
  • Autorisez tous les scopes demandés par GitHub
  • Vérifiez la configuration de votre OAuth App sur GitHub
