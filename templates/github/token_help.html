Configuration du Token GitHub

Token GitHub non configuré ou insuffisant

Votre token GitHub actuel n'a pas les permissions nécessaires pour accéder aux repositories privés.

Étapes pour configurer un Personal Access Token (PAT)

1

Aller sur GitHub

Rendez-vous sur GitHub Settings > Developer settings > Personal access tokens

2

Créer un nouveau token

Cliquez sur "Generate new token" > "Generate new token (classic)"

3

Configurer les permissions

Sélectionnez les scopes suivants :

  • repo - Accès complet aux repositories privés
  • read:org - Lecture des informations d'organisation
  • user:email - Accès aux adresses email
4

Autoriser l'organisation

Si vous travaillez avec des repositories d'organisation (comme hove-io), assurez-vous d'autoriser le token pour cette organisation.

5

Copier le token

Copiez le token généré (il commence par ghp_)

6

Configurer dans GitPulse

Allez dans la configuration GitHub et collez le token dans le champ "Client Secret".
